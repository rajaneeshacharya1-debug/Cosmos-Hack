2-minute pitch script

            

0:00–0:20 — Normal tracking can fail when the lost phone has no internet, is locked, or cannot be reached through a normal app.

0:20–0:45 — Auralis Beacon solves this with a secure recovery bridge. SMS is only the ignition layer; the web server and dashboard handle device state, commands and lifecycle.

0:45–1:20 — I send the boot SMS. The protected OPPO phone activates recovery mode, reports battery/network/location and appears live on this dashboard.

1:20–1:45 — I send Ring Device, Stop Ring, Stop Recovery and Resume Recovery. Each command moves through Pending, Received and Completed.

1:45–2:00 — The MacroDroid bridge proves the concept today. In production, a native Android agent replaces MacroDroid for reliable background control.

MacroDroid setup: exact 5-macro stack
Use this as your live phone checklist. The bridge stays alive after dashboard Stop Recovery.

            
          

          

            
Recovery_ModeBridge/poller active. Dashboard Stop Recovery must keep this true.

            
Live_Tracking_EnabledControls location stream only.

            
Auralis_Ring_EnabledControls repeated ring loop.

            
Auralis_Command_ResponseLatest macro-poll response, for if-block matching.

          

        


        

          

            01
Auralis Boot SMS

            
Trigger
SMS received: #AURALIS-BOOT-4729
Actions
Recovery_Mode=true, Live_Tracking_Enabled=true, Auralis_Ring_Enabled=false, send BOOT URL.
Purpose
Ignites recovery when normal internet tracking is unavailable.

          

          

            02
Auralis Live Tracking Stream

            
Trigger
Regular interval: 3–5 seconds.
Constraints
Recovery_Mode=true AND Live_Tracking_Enabled=true.
Action
Send LIVE URL with battery, latitude, longitude and MOBILE_DATA label.

          

          

            03
Auralis Command Poller FAST

            
Trigger
Regular interval: 1–2 seconds.
Constraint
Recovery_Mode=true.
Initial poll
GET /api/device/oppo-f19/macro-poll and save to Auralis_Command_Response.
If blocks
Use exact contains checks: CMD:STOP_RING;, CMD:RING;, CMD:STOP_RECOVERY;, CMD:RESUME_RECOVERY;, CMD:NORMAL_TRACKING;, CMD:POWER_SAVE;.

          

          

            04
Auralis Ring Loop FAST

            
Triggers
Any Change in Value: Auralis_Ring_Enabled. Also Regular interval every 7–8 seconds.
Constraints
Recovery_Mode=true AND Auralis_Ring_Enabled=true.
Actions
Set media volume high and play /auralis-beacon-alert.wav.

          

          

            05
Auralis Emergency SMS Stop

            
Trigger
SMS received: #AURALIS-STOP-4729
Actions
Recovery_Mode=false, Live_Tracking_Enabled=false, Auralis_Ring_Enabled=false, stop sound, send STOP URL.
Purpose
Full manual shutdown. Dashboard resume requires SMS boot again after this.

          

        


        

          
Rules that prevent sync bugs

          

            
Never set Recovery_Mode=false from dashboard Stop Recovery.That would kill the poller and block Resume Recovery.

            
Use CMD:RING; not loose “contains RING”.STOP_RING contains the word RING and can accidentally trigger the wrong block.

            
Ring Loop uses Auralis_Ring_Enabled, not Auralis_Command_Response.The command response changes back to NONE after polling; it is not a stable state variable.

Server config
MacroDroid Command Poller
Use this as the fourth demo macro. It checks the server for a pending command, runs limited phone actions, then reports completion.
Auralis alert audio/auralis-beacon-alert.wav
PollRegular interval, every 5 seconds while Recovery_Mode = true.
If response contains RINGPlay the Auralis alert sound or speak alert, then call complete URL.
If response contains STOP_RINGStop playback / disable ring loop, then call complete URL.
If response contains STOP_RECOVERYSet Live_Tracking_Enabled false, keep Recovery_Mode true, then call complete URL.
If response contains NORMAL_TRACKING / POWER_SAVE / RESUME_RECOVERYSet Live_Tracking_Enabled true when resuming/normal/power-save, then call complete URL.
