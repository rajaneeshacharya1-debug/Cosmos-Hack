Shared variables

          

            
Recovery_ModeBridge/poller active. Dashboard Stop Recovery keeps this true.

Auralis_Ring_EnabledControls repeated ring loop.

            
Auralis_Command_ResponseLatest macro-poll response used in if-blocks.

          

        


        

          

            01
Auralis Boot SMS

            
Trigger
SMS received: #AURALIS-BOOT-4729
Actions
Set Recovery_Mode=true, Live_Tracking_Enabled=true, Auralis_Ring_Enabled=false, then send BOOT URL.

          

          

            02
Auralis Live Tracking Stream

            
Trigger
Regular interval: 3–5 seconds.
Constraints
Recovery_Mode=true AND Live_Tracking_Enabled=true.
Action
Send LIVE URL with battery, latitude, longitude and MOBILE_DATA.

          

          

            03
Auralis Command Poller FAST

            
Trigger
Regular interval: 1–2 seconds.
Constraint
Recovery_Mode=true.
Initial poll
GET /macro-poll and save response to Auralis_Command_Response.
Conditions
Use exact checks: CMD:STOP_RING;, CMD:RING;, CMD:STOP_RECOVERY;, CMD:RESUME_RECOVERY;, CMD:NORMAL_TRACKING;, CMD:POWER_SAVE;.

          

          

            04
Auralis Ring Loop FAST

            
Triggers
Any Change in Value: Auralis_Ring_Enabled, plus Regular interval every 7–8 seconds.
Constraints
Recovery_Mode=true AND Auralis_Ring_Enabled=true.
Actions
Set media volume high and play auralis-beacon-alert.wav.

          

          

            05
Auralis Emergency SMS Stop

            
Trigger
SMS received: #AURALIS-STOP-4729
Actions
Set Recovery_Mode=false, Live_Tracking_Enabled=false, Auralis_Ring_Enabled=false, stop sound, send STOP URL.

          

        


        

          
Rules that prevent sync bugs

          

            
Dashboard Stop Recovery must not set Recovery_Mode=false.It only pauses Live_Tracking_Enabled so the poller can still resume.

            
Use CMD:RING; not loose RING.STOP_RING contains RING, so loose matching can trigger the wrong block.

            
Ring Loop uses Auralis_Ring_Enabled.Auralis_Command_Response is only the latest poll text, not a stable state.

MacroDroid Command Poller
Use this as the fourth demo macro. It checks the server for a pending command, runs limited phone actions, then reports completion.
Auralis alert audio/auralis-beacon-alert.wav
PollRegular interval, every 5 seconds while Recovery_Mode = true.
If response contains RINGPlay the Auralis alert sound or speak alert, then call complete URL.
If response contains STOP_RINGStop playback / disable ring loop, then call complete URL.
If response contains STOP_RECOVERYSet Live_Tracking_Enabled false, keep Recovery_Mode true, then call complete URL.
If response contains NORMAL_TRACKING / POWER_SAVE / RESUME_RECOVERYSet your tracking variables, then call complete URL.
